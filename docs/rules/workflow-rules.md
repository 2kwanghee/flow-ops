# Workflow Rules

## 자동 작업 기록 (Linear 연동)

코드 작업을 완료한 후 `/log-work` 스킬을 **자율적으로 호출**하여 Linear에 기록한다.

### 자율호출 조건
- 파일 생성/수정/삭제를 포함하는 유의미한 작업을 완료했을 때
- 버그 수정, 기능 구현, 리팩토링, 테스트 추가 등을 마쳤을 때
- 사소한 변경(오타, 포맷팅)은 제외

## TODO.md 기반 작업 추적 (필수)

모든 작업은 프로젝트 루트의 TODO.md를 통해 추적한다. 세션이 중단되어도 다음 세션에서 이어갈 수 있도록 하기 위함.

### 규칙
1. **새 작업 요청 시**: 기존 TODO.md를 초기화하고 새로운 TODO.md를 작성
2. **작업 진행 중**: 완료된 항목은 `[x]`로 체크, 진행 중인 항목은 `[ ]` 유지
3. **세션 시작 시**: TODO.md가 존재하면 먼저 읽고 미완료 작업을 파악하여 이어서 진행
4. **작업 완료 시**: TODO.md의 모든 항목을 `[x]`로 체크

### TODO.md 형식
```markdown
# TODO - [작업 제목]
> 생성일: YYYY-MM-DD
> 상태: 진행중 | 완료

## 목표
[작업의 전체 목표 설명]

## 작업 항목
- [ ] 항목 1
  - [ ] 세부 항목 1-1
- [x] 완료된 항목

## 현재 진행 상황
[마지막으로 진행한 작업 설명]

## 메모
[작업 중 발견한 이슈나 참고사항]
```

## Thinking 로그 (필수)

- 복잡한 작업 수행 시 반드시 **단계별 진행 상황을 텍스트로 출력**한다.
- 사용자가 잘못된 방향으로 가고 있을 때 Escape로 즉시 중단할 수 있도록 각 단계를 명시적으로 표시한다.
- "지금 ~하고 있습니다", "다음으로 ~합니다" 형태로 현재 상태를 알려준다.
